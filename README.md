## Course

CS 300: Data Structures and Algorithms

## Repository Purpose

This repository contains portfolio artifacts completed during CS 300. The artifacts demonstrate my understanding of data structures, algorithms, runtime analysis, and software design.

## Reflection

### What was the problem you were solving in the projects for this course?

The main problem I was solving was how to efficiently organize, store, search, and display course information. Throughout the course I worked with several data structures, including vectors, linked lists, hash tables, and binary search trees. In the final project, I created a program that loads course information from a CSV file, stores the data, prints the courses in alphanumeric order, and allows the user to search for a specific course.

### How did you approach the problem? Consider why data structures are important to understand.

I approached each project by breaking it into smaller tasks before writing code. I first planned how the data would be stored, then determined how users would search for it and how the information would be displayed. Learning multiple data structures helped me understand that each one has strengths and weaknesses. Choosing the correct data structure can improve both the speed and efficiency of a program, making it an important part of software development.

### How did you overcome any roadblocks you encountered while going through the activities or project?

One of the biggest challenges I faced was troubleshooting compilation errors and making sure the program behaved as expected. I learned to debug by testing one function at a time, checking file paths, reading compiler messages carefully, and verifying that each menu option worked correctly. Breaking problems into smaller pieces made them much easier to solve.

### How has your work on this project expanded your approach to designing software and developing programs?

These projects taught me that designing software involves more than simply writing code that works. Planning ahead with pseudocode, organizing code into functions, and selecting the appropriate data structure all contribute to creating a better program. I also gained a better understanding of algorithm efficiency and how different implementations affect performance.

### How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

This course helped me develop better programming habits. I now organize my code into smaller functions with meaningful names instead of placing everything inside the main function. I also pay more attention to readability and writing code that can be modified easily in the future. Overall, I have become much more confident in building programs that are organized, efficient, and easier to maintain.
